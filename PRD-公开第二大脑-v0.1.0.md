# PRD：公开第二大脑 · 个人站

> 版本：v0.1.0 (Demo)  
> 状态：MVP 阶段  
> 底层驱动：Obsidian 知识库 → Markdown → 站点自动同步

---

## 1. Why — 为什么要做

**核心问题**：笔记写了大量有价值的内容，却只躺在 Obsidian 里吃灰。输出倒逼输入是最高效的学习方式，但缺少一个"对外展示的窗口"让知识流动起来。

**战略价值**：
- **输出倒逼输入**：知道有人会看，写笔记的质量自然提升
- **建立影响力**：创业筹备期，内容 = 信任资产，长期积累的笔记就是最好的名片
- **公开记录创业过程**：从零到一折腾个人站、研究产品、踩坑复盘，这些内容本身就是稀缺的

**一句话定位**：一个产品经理创业路上的公开思考集。

## 3. 信息架构

```
首页（自我介绍 + 最新动态 + 入口导航）
├── 关于我（背景 / 价值观 / 目前做的事 / 联系）

│   ├── 创业思考
│   ├── AI 与工具
│   └── 阅读笔记
├── 项目作品（作品集 / 复盘）
└── 订阅（RSS / 邮件订阅入口）
```

## 4. 非目标（Non-goals）

- ❌ 不做登录/注册系统
- ❌ 不做评论区（初期不需要 UGC 管理负担）
- ❌ 不做多语言
- ❌ 不做自定义域名之外的 SEO 优化
- ❌ 不做复杂的 CMS 后台（全靠 Obsidian 管理）

## 5. MVP 范围（Phase 1）

**核心交付**：一个可访问的静态站点，满足"写→发→读"的最小闭环。

| 模块 | 功能 | 说明 |
|------|------|------|
| 首页 | 自我介绍 + 最新笔记列表 + 导航 | 一个页面解决"这人谁 + 写了啥 + 去哪看" |
| 笔记流 | 笔记列表（倒序）+ 标签筛选 | 所有公开笔记按时间排列，可点标签过滤 |
| 单篇笔记 | Markdown 渲染 + 元信息 | 标题 / 日期 / 标签 / 正文 / 返回列表 |
| 关于我 | 个人简介 + 联系方式 | 背景 / 当前创业方向 / 邮箱或微信 |
| 数据接入 | Obsidian 笔记→Markdown→站点 | 自动化/半自动化同步；理解 Obsidian Markdown 语法 |

**MVP 不包含**：主题聚合页、搜索（P1+）、项目作品页（P2）、订阅（P2）、数据统计（P2）

## 9. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 写了没人看 | 中 | 打击动力 | MVP 阶段目标就是让内容先存在，不是追流量 |
| 更新频率跟不上 | 中 | 站点变死站 | 利用 Obsidian 已有笔记做内容冷启动，不要求全部新写 |
| 技术方案选型错 | 低 | 重做 | 静态站是最安全的方案，未来迁移成本极低 |
| 隐私顾虑 | 低 | 心理障碍 | 选"可公开"的笔记发，不把所有 Obsidian 笔记同步上去 |

## 10. 技术方向建议（零代码友好）

鉴于你没有编程背景且不用服务器，推荐路线：

| 方案 | 适合 | 注意 |
|------|------|------|
| **Obsidian 插件 + GitHub Pages** | 纯静态站，自由度中等 | 需要 GitHub 账号，Obsidian 插件（如 Digital Garden）一键发布 |
| **Notion + Super.so** | 最快上线，零代码 | 内容在 Notion 管理，但 Notion 是"另一个库"，和 Obsidian 不打通 |
| **Hugo / Jekyll + GitHub Pages** | 需要我帮你建站 | 模板最多，但需要我一次性帮你配置好 |

**我的推荐**：**方案 1（Obsidian 插件 + GitHub Pages）**。因为：
1. 你的内容源头就是 Obsidian，不需要额外迁移
2. GitHub Pages 完全免费、无服务器
3. Obsidian 社区有成熟的发布插件（Digital Garden / Obsidian Publish 替代品）
4. 我可以帮你一次性搭好，以后你只需要在 Obsidian 写，同步发布
