一人创业和团队创业最大的区别不是人数，是**决策半径**。

团队创业你有合伙人、投资人、顾问——信息面广，但决策慢。一人创业你说了算，决策快，但信息面窄，容易走偏。

## 我的一人创业过滤框架

```
需求 → 验证 → 方案 → 验证 → 产品 → 验证
```

每个箭头中间都插一次验证，这是在"信息面窄"的情况下控制风险的方式。

### 第一层：需求验证

不要想"这个功能很酷"，要想"谁需要这个、他愿不愿意为此付费"。

用猪八戒网、淘宝服务市场、知乎去看看有没有人在找这类服务——有人找 = 有需求。

### 第二层：方案验证

有了需求之后，不要直接开干。先找**最小的可卖方案**：
- 一个 LP 页 + 付款链接
- 一篇讲清楚"我做什么"的文章
- 甚至一条朋友圈

有人愿意付款，才进入正式开发。

### 第三层：产品验证

MVP 上线后关注两件事：① 用户能不能用起来 ② 用户愿不愿意留下来。

如果第一版上线两周没有自然留存，要么需求不对，要么体验太差。

## 核心原则

一人创业的容错率很低。一个错误判断可能浪费几个月。

所以原则是：**多验证，少假设。多做小实验，少上大项目。**
