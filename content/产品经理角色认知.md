# 产品经理角色认知

## 产品经理是什么

产品经理（PM）是产品的**定义者、推动者、守护者**。负责发现用户需求，定义产品方案，协调各方资源，推动产品从概念到交付的全过程。

## 核心职责

| 职责 | 说明 |
|------|------|
| **需求管理** | 收集、分析、定义、优先级排序 |
| **产品规划** | 制定路线图、版本规划、里程碑 |
| **文档输出** | PRD、原型、用户手册 |
| **项目推进** | 协调设计/开发/测试，确保按时交付 |
| **数据分析** | 监控数据、分析行为、驱动迭代 |
| **用户洞察** | 用户研究、反馈收集、体验优化 |

## 能力模型

### 硬技能
- 需求分析与文档能力
- 数据分析能力
- 项目管理能力
- 商业建模能力

### 软技能
- 沟通与表达能力
- 同理心与用户思维
- 逻辑思维与批判性思维
- 学习能力与好奇心

### 行业知识
- 所在行业的垂直知识
- 技术基本认知（理解开发实现）
- 设计基本认知（审美与可用性）

## 成长路径

```
产品助理 → 产品经理 → 高级产品经理 → 产品总监 → CPO/VP Product
```

- **0-1 年**：熟悉流程，执行落地
- **1-3 年**：独立负责模块，建立方法论
- **3-5 年**：负责整条产品线，带新人
- **5-8 年**：产品总监，制定产品战略
- **8 年+**：CPO，业务操盘手
